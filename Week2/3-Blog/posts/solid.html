SOLID Principles: Strong Foundations in Software Design

Introduction

The SOLID principles provide guidance for creating high-quality, maintainable, and scalable software architecture. These principles help manage software component dependencies and promote flexible and easily extensible structures.

Loosely Coupled: Managing Dependencies

Objects that are tightly coupled affect software flexibility. For instance, directly instantiating a class inside another class leads to tight coupling, making changes more difficult.

class MailSender {
    public void Send(IMailServer mailServer) {
        mailServer.Send("nazif@karaca.com", "Hello World!");
    }
}

Single Responsibility Principle (SRP)

A class should have only one reason to change, meaning it should have only one responsibility.

class Database {
    public void Connect() {
        Console.WriteLine("Database connected.");
    }
}

class PersonService {
    public List<Person> GetPersons() {
        return new() {
            new() { Name = "Nazif", Surname = "Karaca" },
            new() { Name = "Eren", Surname = "Öztürk" }
        };
    }
}

Open/Closed Principle (OCP)

Software entities should be open for extension but closed for modification.

interface IBank {
    bool MoneyTransfer(int amount, string accountNo);
}

Conclusion

SOLID principles ensure sustainable, flexible, and modular software development. Implementing these principles makes your software more readable, testable, and extensible.
